Benutzerhandbuch zum Datenimport

Benutzerversion
▾

Sie können Daten Dateien von Ihrem Computer über das lokale Netzwerk in TimeTrails importieren.

Der gesamte Prozess Kein Datenkabel erforderlich , auch Kein Konto-Login erforderlich .

1. Vorbereitungsarbeiten

✓
Verbinden Sie Ihr Telefon und Ihren Computer mit demselben Wi-Fi
✓
Bereiten Sie die Datei für den Import vor
Unterstützt die Datenbankdatei .db und die Trackdatei .gpx

2. Importdienst auf Mobilgeräten aktivieren

  1. 1
    Öffnen Sie TimeTrails
  2. 2
    Gehen Sie zur Seite 'Datenimport / Datentransfer'
  3. 3
    Klicken Sie auf 'Dienst starten'
  4. 4
    Nach dem Erfolg wird eine URL (lokale Netzwerkadresse) auf der Seite angezeigt.
    Zum Beispiel: http://192.168.1.100:9527

3. Dateien auf dem Computer hochladen

  1. 1
    Geben Sie im Browser Ihres Computers die URL ein, die auf Ihrem Telefon angezeigt wird.
  2. 2
    Öffnen Sie es, und wählen Sie dann Ihre Datei aus und laden Sie sie hoch.
  3. 3
    Nach dem Abschluss des Uploads wird die Datei automatisch in der Dateiliste auf Ihrem Telefon angezeigt.

4. Daten importieren

In der Dateiliste auf Ihrem Telefon:

.db Datenbankdatei

Wird verwendet, um gesamte TimeTrails-Daten wiederherzustellen oder zu migrieren

  1. 1
    Klicken Sie auf die .db-Datei
  2. 2
    Wählen Sie 'Importieren'
  3. 3
    Eine Erfolgsmeldung wird nach Abschluss angezeigt.

.gpx Track-Datei

Nach dem Abschluss des Imports nach Datum auf der Karte angezeigt.

  1. 1
    Klicken Sie auf die .gpx-Datei
  2. 2
    Das System wird die Strecke automatisch analysieren und importieren.
  3. 3
    Überprüfen Sie die Karte nach Abschluss
⚠️ Bitte halten Sie die App während des Importvorgangs im Vordergrund, und Sie werden eine Erfolgsmeldung sehen, sobald der Vorgang abgeschlossen ist.

5. Häufige Tipps und Erklärungen

!
Dienst kann nicht gestartet werden
Bitte stellen Sie sicher, dass Sie mit Wi-Fi und nicht mit dem Mobilfunknetz verbunden sind.
!
Import fehlgeschlagen oder einige Daten wurden übersprungen
Einige Streckenpunkte fehlen an Zeitinformationen, und das System wird sie automatisch ignorieren.
!
Datei nicht gefunden
Aktualisieren Sie die Dropdown-Liste, um erneut zu scannen.

6. Einige Vorschläge

✓
Vor dem Import wird empfohlen, die aktuellen Daten zu sichern.
✓
.db Dateien werden typischerweise für die vollständige Migration verwendet.
✓
.gpx Eher geeignet für den Import von einzelnen oder mehreren Streckenaufzeichnungen
